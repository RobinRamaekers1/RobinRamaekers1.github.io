Project 4.0
Doel:

Het doel van dit project was een crowd monitoring systeem maken voor het bedrijf Mediaan.

Werking:

In het vak Project4.0 hebben we de opportuniteit gekregen om een opdracht te maken voor Mediaan. De opdracht was een crowd monitoring systeem maken met de bedoeling om de veiligheid van evenementen zoals voetbal, festivals, etc. te verbeteren. Dit hebben we gedaan met behulp van Artificiële intelligentie. Medestudent Lenn Van Genechten en ik stonden in voor het AI-gedeelte, Jari Van De Vel en Renzo Beekmans stonden in voor de front-en backend van de webapplicatie. Jesse Dierx en Tim Verbecque zorgde ervoor dat de security en pipelines op orde stonden, alsook dat alle componenten synchroon met elkaar werkten. In het AI-gedeelte hebben we 2 Deep Learning modellen ontworpen, één model stond in om de gemoedstoestand te herkennen (neutraal, negatief en positief) de andere stond in om diverse activiteiten zoals vuurwerk, donder, applaus, sirene, etc. Het geluid wordt live opgenomen in segmenten van 5 seconden door middel van een microfoon. Na 5 seconden wordt het bestand opgeslagen en door de AI-modellen geanalyseerd. De resultaten worden opgeslagen in de databank en naar de web app doorgestuurd zodat de security deze kan analyseren en backup kan sturen naar de doorgegeven locatie.

conclusie:

Deze proof of concept was een zeer leuke ervaring, die mijn kennis zeer hard op de proef heeft gesteld. Ik heb veel bijgeleerd over de verschillende technieken die je kan gebruiken als je Python en audio wilt gebruiken om een accuraat AI-model te maken. Het was ook zeer leuk om alle domeinen van IT in één groep te hebben omdat in een conventioneel groepsproject je alleen maar zit met studenten die dezelfde richting volgen. Ook heb ik geleerd hoeveel werk en tijd erin steekt voordat je daadwerkelijk naar de realisatiefase kan gaan. Hiervoor dank aan Mediaan en Thomas More om ons deze mogelijkheid en ervaring te geven.

Artificial Intelligence Project
Doel:

Een auto zelf een parkour laten rijden in de Udacity simulator.

Werking:

In dit groepsproject heb ik samen met medestudenten Stan Swinnen en Yunus Kurban een zelfrijdende auto gemaakt in de Udacity Driving simulator. We zijn dit project begonnen met een OpenCV implementatie, dit houdt in dat het middelpunt van het scherm wordt berekend in realtime adhv. deze calculaties kan de stuurrichting berekend worden. Dit hebben we gedaan om een inzicht te krijgen over hoe de simulator werkt. Daarna zijn we van start gegaan met de simulator zelf, de eerste stap was om training data te verzamelen hiervoor was er in ingebouwde tool. Na enkele rondes voor -en achterwaarts te rijden terwijl er training data opgenomen wordt, is het tijd om het model te trainen, kon de auto met gemak het hele parkour meerdere keren afleggen. In dit project ben ik bezig geweest met de OpenCV optimalisatie en de juiste keybindings op de bijhorende acties zetten zodat de auto kan rijden alsook de training data en debugging van het Deep Learning model.

Conclusie:

Dit project was redelijk uitdagend omdat we maar één maand hadden om het uit te voeren, dit zorgde ervoor dat we heel gestructureerd moesten werken om elke week een nieuw deel werkend te krijgen. Ik heb hier veel bijgeleerd over OpenCV en edge detection, je moet veel experimenteren met de parameters om de beste resultaten te krijgen. Ik ben er ook achtergekomen dat OpenCV een sterke tool is maar dat dit zijn limitaties heeft tegenover Deep Learning technieken. Als laatste kan ik ook nog aanraden om genoeg data te verzamelen als je een Deep Learning project zoals dit wilt maken, dit elimineert de kans op fouten bij de moeilijke stukken van het parkour.

November 2022 - December 2022

Big Data
Doel:

In dit groepsproject was het de bedoeling om een Deep Learning model te maken met gebruik van FastAI. We hebben gekozen om fietsen te gaan classificeren omdat dit zeer uitdagend kan zijn omdat sommige types van fietsen zeer sterk op elkaar lijken. Het einddoel van dit project was om Google Teachable Machine te verslaan in accuraatheid.

Werking:

In dit groepsproject heb ik samen met medestudenten Britt De Keyser en Syan Delbart een Deep Learning model gemaakt dat in staat is om verschillende types fietsen te classificeren zoals: Mountainbike, Eroica bike, Wielrenfiets, Gravel bike en Speed pedilac. De eerste stap was om een webscraper te ontwikkelen die op een gebruiksvriendelijke manier deze foto`s kan verzamelen. Hierna moest er een grondige EDA gebeuren op de data omdat er veel foto`s waren die we niet konden gebruiken, denk aan mensen die langs de fiets staan en fietsonderdelen. Hierna werd er een model getraind gebaseerd op een Restnet 50 algoritme zonder fine-tuning. Dit gaf ons een accuraatheid van 69 percent. We vonden dit nog niet genoeg dus we hebben een geavanceerd model gemaakt op basis van het Restnet 50 model, dit gaf ons een accuraatheid van 76 percent. Dit model was ook in staat om Google Teachable Machine te verslaan. De laatste stap was om ons model te implementeren in een Streamlit omgeving om de gebruikservaring te verbeteren. In dit project ben ik bezig geweest met de data collection en cleaning van de mountainbike en wielrenfiets foto`s. Ik heb ook het advanced model en de Streamlit omgeving gemaakt.

Conclusie:

Dit project was heel interessant omdat we hier met de zogenaamde "state of the art" Deep Learning technieken hebben gebruikt. Het was ook zeer interessant om een webscraper te maken om de data collectie makkelijker te laten gaan. Ook heb ik geleerd dat Streamlit een zeer krachtige manier is om de predicties van je AI-modellen te visualiseren.
